# A02: Security Misconfiguration — セキュリティ設定ミス

## 対象ラボ

### 1. デフォルト認証情報

- **概要**: admin/admin 等のデフォルトパスワードでログイン可能
- **攻撃例**: よくある認証情報の辞書を使ってログイン試行
- **技術スタック**: Hono API + PostgreSQL
- **難易度**: ★☆☆

### 2. 不要なHTTPメソッド許可

- **概要**: PUT / DELETE / TRACE 等を制限なく受け付ける
- **攻撃例**: `curl -X DELETE /api/users/1` で意図しないリソース削除
- **技術スタック**: Hono API
- **難易度**: ★☆☆

### 3. ディレクトリリスティング

- **概要**: ファイル一覧がブラウザから閲覧可能
- **攻撃例**: `/static/` にアクセスしてサーバー上のファイル一覧を取得
- **技術スタック**: Hono 静的ファイル配信
- **難易度**: ★☆☆

### 4. 詳細エラーメッセージ露出

- **概要**: スタックトレースやDB構造がエラーレスポンスに含まれる
- **攻撃例**: 不正なリクエストを送り、レスポンスからテーブル名やカラム名を取得
- **技術スタック**: Hono API + PostgreSQL
- **難易度**: ★☆☆

### 5. 不要なヘッダー露出

- **概要**: X-Powered-By 等でサーバー技術情報が漏洩
- **攻撃例**: レスポンスヘッダーからフレームワーク・バージョンを特定
- **技術スタック**: Hono API
- **難易度**: ★☆☆
